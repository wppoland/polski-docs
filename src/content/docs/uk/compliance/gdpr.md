GDPR (RODO) вимагає від магазинів отримання чіткої згоди на обробку персональних даних. Плагін додає 7 налаштовуваних чекбоксів на сторінці замовлення, логування згод та інструменти управління згодами.

## Необхідні згоди у польському e-commerce

Магазин повинен збирати згоди на:

1. Прийняття регламенту магазину
2. Ознайомлення з політикою конфіденційності
3. Право на відмову від договору (підтвердження ознайомлення)
4. Згода на надання цифрового контенту до закінчення терміну відмови
5. Сповіщення про доставку (SMS/e-mail)
6. Нагадування про відгук
7. Маркетинг (розсилка, комерційні пропозиції)

## Налаштування чекбоксів

Перейдіть до **WooCommerce > Налаштування > Polski > RODO** та налаштуйте згоди.

### 1. Регламент магазину

Обов'язковий чекбокс з посиланням на сторінку регламенту.

| Налаштування | Опис |
|------------|------|
| Текст | Налаштовуваний, за замовчуванням: "Zapoznałem/am się z [regulaminem] i akceptuję jego warunki" |
| Обов'язковий | Так (завжди) |
| Сторінка регламенту | Виберіть зі сторінок WordPress |

### 2. Політика конфіденційності

Обов'язковий чекбокс з посиланням на політику конфіденційності.

| Налаштування | Опис |
|------------|------|
| Текст | За замовчуванням: "Zapoznałem/am się z [polityką prywatności]" |
| Обов'язковий | Так (завжди) |
| Сторінка політики | Виберіть зі сторінок WordPress |

### 3. Право на відмову від договору

Інформація про ознайомлення з умовами відмови.

| Налаштування | Опис |
|------------|------|
| Текст | За замовчуванням: "Zapoznałem/am się z warunkami [odstąpienia od umowy]" |
| Обов'язковий | Так |
| Сторінка відмови | Виберіть зі сторінок WordPress |

### 4. Цифровий контент

Згода, необхідна при продажу цифрового контенту (наприклад, e-book, файли для завантаження).

| Налаштування | Опис |
|------------|------|
| Текст | За замовчуванням: "Wyrażam zgodę na dostarczenie treści cyfrowych przed upływem terminu do odstąpienia od umowy i przyjmuję do wiadomości utratę prawa do odstąpienia" |
| Обов'язковий | Так (коли кошик містить цифрові продукти) |
| Умова | Відображати лише коли кошик містить віртуальні або завантажувані продукти |

### 5. Сповіщення про доставку

Згода на надсилання SMS/e-mail сповіщень про статус відправлення.

| Налаштування | Опис |
|------------|------|
| Текст | За замовчуванням: "Wyrażam zgodę na otrzymywanie powiadomień o statusie dostawy" |
| Обов'язковий | Ні |
| Канал | E-mail, SMS або обидва |

### 6. Нагадування про відгук

Згода на надсилання e-mail з проханням залишити відгук після покупки.

| Налаштування | Опис |
|------------|------|
| Текст | За замовчуванням: "Wyrażam zgodę na otrzymanie e-maila z prośbą o wystawienie opinii o zakupionym produkcie" |
| Обов'язковий | Ні |
| Затримка | Кількість днів після доставки (за замовчуванням 7) |

### 7. Маркетинг

Згода на маркетингову комунікацію.

| Налаштування | Опис |
|------------|------|
| Текст | За замовчуванням: "Wyrażam zgodę na otrzymywanie informacji handlowych drogą elektroniczną" |
| Обов'язковий | Ні |
| Обсяг | Розсилка, пропозиції, акції |

## Логування згод

Кожна згода записується в базу даних з даними:

| Поле | Опис |
|------|------|
| ID користувача | ID клієнта WordPress (або 0 для гостей) |
| ID замовлення | Номер замовлення WooCommerce |
| Тип згоди | Ідентифікатор чекбоксу (наприклад, `terms`, `privacy`, `marketing`) |
| Значення | `granted` або `denied` |
| IP-адреса | Анонімізована IP-адреса клієнта |
| User Agent | Браузер та операційна система |
| Мітка часу | Дата та час надання згоди (UTC) |
| Версія документа | Хеш версії регламенту/політики на момент надання згоди |

### Перегляд логів згод

Логи згод доступні у:

- **Замовлення WooCommerce** - вкладка "Згоди RODO" у бічній панелі замовлення
- **Профіль користувача** - розділ "Історія згод" у профілі клієнта в адміністративній панелі
- **Експорт** - можливість експорту логів до CSV (**WooCommerce > Налаштування > Polski > RODO > Експорт логів**)

### Анонімізація IP

Плагін анонімізує останній октет IPv4 (наприклад, `192.168.1.xxx`) та останню групу IPv6. Це забезпечує відповідність GDPR та зберігає базову корисність логів.

## Шорткод API

### Відображення статусу згод клієнта

```
[polski_consent_status]
```

Показує авторизованому клієнту список згод з можливістю їх відкликання (наприклад, маркетингова згода).

### Форма відкликання маркетингової згоди

```
[polski_consent_withdraw type="marketing"]
```

Форма для відкликання маркетингової згоди. Після відкликання плагін автоматично оновлює статус згоди в базі даних.

### Параметри шорткоду

| Параметр | Опис | Доступні значення |
|----------|------|-------------------|
| `type` | Тип згоди | `terms`, `privacy`, `withdrawal_right`, `digital_content`, `delivery_notifications`, `review_reminder`, `marketing` |

## Інтеграція з WooCommerce Blocks

Чекбокси згод працюють також з блочною формою замовлення (WooCommerce Blocks Checkout). Нічого не потрібно налаштовувати.

## Право бути забутим

Плагін працює з інструментом WordPress **Інструменти > Видалити персональні дані**. Після затвердження запиту плагін автоматично:

1. Анонімізує дані у логах згод
2. Видаляє персональні дані з форм відмови
3. Зберігає анонімізовані записи для цілей звітності

## Право на переносимість даних

Плагін працює з **Інструменти > Експорт персональних даних**. Експорт містить:

- Історію наданих згод
- Дані з форм (анонімізовані)
- Комунікаційні вподобання

## Вирішення проблем

**Чекбокси не відображаються на сторінці замовлення**
Перевірте, чи модуль RODO увімкнений у **WooCommerce > Налаштування > Polski > Модулі**. При блочній формі замовлення потрібен WooCommerce 8.0+.

**Клієнт не може оформити замовлення**
Перевірте, чи інший плагін (наприклад, Germanized, WPML) не додає ті ж чекбокси. Вимкніть згоди з інших плагінів та використовуйте лише модуль Polski for WooCommerce.

**Логи згод не записують IP-адресу**
Перевірте, чи сервер передає IP-адресу. За reverse proxy (наприклад, Cloudflare) налаштуйте заголовок `X-Forwarded-For` у WordPress.
