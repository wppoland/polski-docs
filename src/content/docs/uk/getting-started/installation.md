## Попередні вимоги

Перевірте, чи ваш сервер відповідає вимогам. Плагін **Polski for WooCommerce** 1.3.2 вимагає:

| Компонент | Мінімальна версія | Рекомендована версія |
|-----------|-----------------|-----------------|
| WordPress | 6.4+ | 6.7+ |
| WooCommerce | 8.0+ | 9.x |
| PHP | 8.1+ | 8.2+ |
| MySQL | 5.7+ | 8.0+ |
| MariaDB | 10.3+ | 10.11+ |

### Перевірка версії PHP

Не знаєте версію PHP? Перевірте в панелі WordPress:

1. Перейдіть до **Інструменти > Здоров'я сайту**
2. Натисніть вкладку **Інформація**
3. Розгорніть розділ **Сервер**
4. Знайдіть поле **Версія PHP**

Версію PHP також знайдете в панелі хостингу (cPanel, DirectAdmin) в налаштуваннях домену.

### Перевірка версії WooCommerce

1. Перейдіть до **Плагіни > Встановлені плагіни**
2. Знайдіть **WooCommerce** у списку
3. Версія відображається під назвою плагіна

:::caution[PHP 8.0 та старіші]
Плагін не працює на PHP 7.x та PHP 8.0. Якщо ваш хостинг не має PHP 8.1+, змініть версію PHP в панелі хостингу або зверніться до провайдера.
:::

## Метод 1: Встановлення з панелі WordPress (рекомендований)

Це найпростіший та рекомендований метод встановлення.

### Крок 1 - відкрийте панель плагінів

Увійдіть в адміністративну панель WordPress і перейдіть до **Плагіни > Додати новий плагін**.

### Крок 2 - знайдіть плагін

У полі пошуку введіть:

```
Polski for WooCommerce
```

Плагін повинен з'явитися як перший результат. Автором є **wppoland.com**.

### Крок 3 - встановіть плагін

Натисніть кнопку **Встановити зараз** біля плагіна "Polski for WooCommerce". Зачекайте, поки WordPress завантажить та розпакує файли.

### Крок 4 - активуйте плагін

Після завершення встановлення кнопка зміниться на **Активувати**. Натисніть її, щоб активувати плагін.

### Крок 5 - перейдіть до налаштування

Після активації з'явиться сповіщення з посиланням на майстер налаштування. Натисніть **Налаштувати плагін** або перейдіть вручну до **WooCommerce > Polski**.

## Метод 2: Ручне встановлення з ZIP-файлу

Використовуйте цей метод, коли хочете встановити конкретну версію або сервер не з'єднується з WordPress.org.

### Крок 1 - завантажте ZIP-пакет

Завантажте останню версію плагіна з одного з наступних джерел:

- **WordPress.org**: [https://wordpress.org/plugins/polski-for-woocommerce/](https://wordpress.org/plugins/polski-for-woocommerce/)
- **GitHub Releases**: [https://github.com/wppoland/polski/releases](https://github.com/wppoland/polski/releases)

### Крок 2 - завантажте ZIP-файл

1. Перейдіть до **Плагіни > Додати новий плагін**
2. Натисніть кнопку **Завантажити плагін** вгорі сторінки
3. Натисніть **Вибрати файл** та вкажіть завантажений ZIP-файл
4. Натисніть **Встановити зараз**

### Крок 3 - активуйте плагін

Після завантаження та розпакування файлів натисніть **Активувати плагін**.

## Метод 3: Встановлення через FTP/SFTP

Використовуйте цей метод, коли панель WordPress не дозволяє завантажувати файли (наприклад, через обмеження розміру).

### Крок 1 - розпакуйте архів

Розпакуйте завантажений ZIP-файл на своєму комп'ютері. Ви отримаєте папку `polski-for-woocommerce`.

### Крок 2 - завантажте на сервер

Підключіться до сервера через FTP/SFTP та завантажте всю папку `polski-for-woocommerce` до каталогу:

```
/wp-content/plugins/
```

Структура після завантаження повинна виглядати так:

```
wp-content/
  plugins/
    polski-for-woocommerce/
      polski-for-woocommerce.php
      includes/
      assets/
      ...
```

### Крок 3 - активуйте в панелі

Перейдіть до **Плагіни > Встановлені плагіни** в панелі WordPress і натисніть **Активувати** біля "Polski for WooCommerce".

## Встановлення через WP-CLI

Якщо маєте доступ до терміналу сервера, встановіть плагін однією командою:

```bash
wp plugin install polski-for-woocommerce --activate
```

Для оновлення до останньої версії:

```bash
wp plugin update polski-for-woocommerce
```

Перевірка поточної версії:

```bash
wp plugin get polski-for-woocommerce --fields=name,version,status
```

## Перевірка встановлення

Після активації плагіна перевірте, чи все працює коректно:

1. **Новий пункт меню** - в меню WooCommerce повинен з'явитися елемент **Polski**
2. **Панель відповідності** - перейдіть до **WooCommerce > Polski**, щоб побачити головну панель
3. **Відсутність помилок** - перевірте, чи на сторінці не з'явилися повідомлення про помилки PHP

### Перевірка за допомогою шорткоду

Щоб перевірити, чи плагін активний, вставте шорткод на будь-якій сторінці:

```
[polski_version]
```

Шорткод покаже версію плагіна (наприклад, "1.3.2") - це означає, що встановлення працює.

## Вирішення проблем

### Плагін не з'являється в пошуку

- Перевірте, чи ваш WordPress має з'єднання з репозиторієм WordPress.org
- Переконайтеся, що шукаєте саме "Polski for WooCommerce"
- Спробуйте ручне встановлення з ZIP-файлу

### Помилка "Плагін вимагає новішої версії PHP"

Ваш сервер використовує непідтримувану версію PHP. Рішення:

1. Увійдіть до панелі хостингу
2. Знайдіть налаштування PHP (зазвичай у розділі "Домени" або "Хостинг")
3. Змініть версію PHP на 8.1 або новішу
4. Зачекайте кілька хвилин на застосування змін
5. Спробуйте повторно активувати плагін

### Помилка "WooCommerce є обов'язковим"

Плагін вимагає активного WooCommerce версії 8.0 або новішої:

1. Переконайтеся, що WooCommerce встановлений та активний
2. Оновіть WooCommerce до останньої версії
3. Повторно активуйте плагін Polski for WooCommerce

### Білий екран після активації

Якщо після активації бачите білий екран:

1. Увімкніть режим налагодження WordPress - додайте до `wp-config.php`:

```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);
```

2. Перевірте файл логів: `wp-content/debug.log`
3. Якщо проблема пов'язана з конфліктом з іншим плагіном, деактивуйте решту плагінів та активуйте їх по одному

### Проблеми з правами доступу до файлів

Якщо ручне встановлення не працює, перевірте права доступу:

```bash
# Рекомендовані права для каталогів
chmod 755 wp-content/plugins/polski-for-woocommerce/

# Рекомендовані права для файлів
chmod 644 wp-content/plugins/polski-for-woocommerce/*.php
```

## Оновлення плагіна

Плагін оновлюється через стандартний механізм WordPress. Коли з'явиться нова версія:

1. Ви побачите сповіщення у **Плагіни > Встановлені плагіни**
2. Натисніть **Оновити зараз** або скористайтеся масовим оновленням
3. Після оновлення перевірте, чи магазин працює коректно

:::tip[Резервна копія]
Перед оновленням зробіть резервну копію бази даних та файлів. Хостинг часто робить це автоматично, але ручна копія дає впевненість.
:::
