Польське законодавство зобов'язує інтернет-продавців однозначно інформувати про те, чи містить вказана ціна податок ПДВ. Плагін Polski for WooCommerce дозволяє гнучко керувати способом відображення інформації про ПДВ - від простого позначення "брутто/нетто" до повної інформації про ставку податку та підстави для звільнення.

## Правові вимоги

Відповідно до Закону про інформування про ціни товарів та послуг і Закону про ПДВ, інтернет-магазин повинен:

- чітко інформувати, чи включає ціна податок ПДВ
- вказувати ставку ПДВ, якщо магазин продає як фізичним особам, так і компаніям
- у випадку звільнення від ПДВ - вказати правову підставу звільнення

Продавці, які користуються суб'єктним звільненням (ст. 113 Закону про ПДВ), повинні повідомити клієнта, що ціна не містить ПДВ через звільнення.

## Налаштування

Перейдіть до **WooCommerce > Налаштування > Polski > Ціни** та налаштуйте розділ "Відображення ПДВ".

### Режими відображення

| Режим | Опис | Приклад |
|-------|------|---------|
| Брутто (з ПДВ) | Ціна включає податок | 123,00 zł брутто |
| Нетто (без ПДВ) | Ціна без податку | 100,00 zł нетто |
| Обидві | Обидві ціни одночасно | 100,00 zł нетто (123,00 zł брутто) |

### Детальні налаштування

- **Показати ставку ПДВ** - відображає процентну ставку податку поряд з ціною (наприклад, "в тому числі 23% ПДВ")
- **Показати інформацію про ПДВ у лістингу** - керує видимістю на сторінках категорій та результатах пошуку
- **Показати інформацію про ПДВ у кошику** - керує видимістю у кошику та підсумку замовлення
- **Власний текст** - дозволяє замінити текст інформації про ПДВ за замовчуванням

## Звільнення від ПДВ (ст. 113)

Продавці, звільнені від ПДВ на підставі ст. 113 п. 1 або п. 9 Закону про податок на товари та послуги, можуть налаштувати відповідне повідомлення.

### Налаштування звільнення

1. Перейдіть до **WooCommerce > Налаштування > Polski > Ціни**
2. Позначте опцію **Звільнення від ПДВ (ст. 113)**
3. Виберіть підставу звільнення:
   - **Ст. 113 п. 1** - звільнення для продажу до 200 000 zł на рік
   - **Ст. 113 п. 9** - звільнення для платників податків, що розпочинають діяльність протягом року
4. За бажанням налаштуйте текст повідомлення

Повідомлення за замовчуванням: "Ціна не містить податку ПДВ - продавець користується звільненням на підставі ст. 113 п. 1 Закону про ПДВ."

### Вимкнення ПДВ у WooCommerce

При звільненні від ПДВ встановіть у WooCommerce:

1. **WooCommerce > Налаштування > Податки** - вимкніть розрахунок податків АБО встановіть ставку 0%
2. Плагін автоматично додасть відповідну примітку до цін

## Shortcode

Використовуйте shortcode `[polski_tax_notice]`, щоб відобразити інформацію про ПДВ у будь-якому місці.

### Параметри

| Параметр | Тип | За замовчуванням | Опис |
|----------|-----|------------------|------|
| `product_id` | int | поточний | ID продукту |
| `type` | string | `auto` | Тип інформації: `auto`, `gross`, `net`, `exempt` |
| `show_rate` | bool | `true` | Чи відображати процентну ставку |
| `wrapper` | string | `span` | HTML-елемент обгортки |

### Приклади використання

Автоматичне визначення на сторінці продукту:

```html
[polski_tax_notice]
```

Примусове відображення інформації про звільнення:

```html
[polski_tax_notice type="exempt"]
```

Без процентної ставки:

```html
[polski_tax_notice show_rate="false"]
```

У шаблоні PHP:

```php
echo do_shortcode('[polski_tax_notice product_id="' . $product->get_id() . '"]');
```

## Налаштування для магазинів B2B та B2C

Магазини, що обслуговують як фізичних осіб (B2C), так і корпоративних клієнтів (B2B), можуть налаштувати різні варіанти відображення цін залежно від ролі користувача.

### Ціни нетто для компаній

Плагін працює з системою ролей WooCommerce. Щоб відображати ціни нетто для корпоративних клієнтів:

1. Створіть окрему роль (наприклад, "klient_firmowy") або використовуйте існуючу
2. У налаштуваннях плагіна призначте відображення нетто для обраної ролі
3. Корпоративні клієнти бачитимуть ціни без ПДВ, а фізичні особи - з ПДВ

### Подвійні ціни на сторінці продукту

Активація режиму "Обидві" відображає ціну нетто та брутто одночасно. Формат відображення:

```
100,00 zł нетто
123,00 zł брутто (в тому числі 23% ПДВ)
```

Порядок та формат можна налаштувати в параметрах.

## Ставки ПДВ для різних категорій продуктів

У Польщі діють чотири ставки ПДВ:

| Ставка | Застосування |
|--------|-------------|
| 23% | Основна ставка - більшість товарів та послуг |
| 8% | Знижена ставка - житлове будівництво, послуги громадського харчування |
| 5% | Знижена ставка - продукти харчування, книги, періодичні видання |
| 0% | Нульова ставка - експорт, внутрішньоспільнотна поставка товарів |

Плагін автоматично розпізнає ставку, призначену продукту у WooCommerce, та відображає відповідну інформацію.

## Інформація про ПДВ у електронних листах

Плагін додає інформацію про ПДВ також до транзакційних електронних листів WooCommerce:

- підтвердження замовлення
- рахунок-фактура
- зміна статусу замовлення

Текст повідомлення відповідає налаштуванням магазину.

## Стилізація CSS

Інформація про ПДВ обгортається елементами з відповідними CSS-класами:

```css
.polski-tax-notice {
    font-size: 0.85em;
    color: #666;
}

.polski-tax-notice--exempt {
    color: #c00;
    font-weight: 600;
}

.polski-tax-notice__rate {
    white-space: nowrap;
}
```

## Найпоширеніші проблеми

### Інформація про ПДВ відображається двічі

Перевірте, чи тема не додає власну інформацію про ПДВ. Деякі теми, призначені для польського ринку, мають вбудовану підтримку ПДВ - у такому випадку вимкніть одну з реалізацій.

### Ставка ПДВ відображається неправильно

Переконайтеся, що податкові класи в **WooCommerce > Налаштування > Податки > Стандартні ставки** налаштовані правильно. Плагін зчитує ставку безпосередньо з конфігурації WooCommerce.
